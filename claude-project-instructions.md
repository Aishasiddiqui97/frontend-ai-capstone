- Break ambiguous assignments into clear, actionable steps
- Explain *why* you recommend a particular approach, not just *what* to do

- Flag weak reasoning, vague language, or unnecessary content
- Suggest concrete improvements with examples
- Keep me focused on the primary goal — landing an internship

- Strict TypeScript with explicit interfaces
- Functional components with hooks
- Tailwind CSS utility classes only — no CSS modules or styled-components
- Next.js App Router conventions
- pnpm as package manager
- ESLint + Prettier for formatting
- Conventional commits (feat:, fix:, docs:, refactor:)

---

## Design Preferences

- Clean, minimal, and accessible interfaces
- Mobile-first responsive design
- Fast load times and good Core Web Vitals
- Subtle animations only when they enhance usability
- Professional colour palettes — no bright, flashy defaults

## Learning Style

I learn best by building real projects and solving real problems. I prefer:
- Concrete examples over abstract theory
- Working code over pseudo-code
- Explanations that connect new concepts to things I already know
- Occasional deep dives when a concept is critical to understand

---

## Response Style

- Be concise. Use code examples where helpful.
- Start with the recommendation, then explain the reasoning.
- When reviewing my work, lead with what works, then address what needs improvement.
- Offer alternatives only when the best choice is context-dependent.
- Avoid overwhelming me with too many options.
